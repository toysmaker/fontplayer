# 可变字体：轴标签（tag）与展示名称

## `wght` 是什么？

OpenType 注册轴 **`wght`** 表示 **字重（Weight）**，数值习惯上对齐 CSS/OS/2 的粗细标度（例如 100–900）。它**不是**「单根笔画的线宽」几何量；宿主应用里若出现「直线宽度」等说法，多为该产品对同一根 `wght` 轴的本地化文案，不代表二进制里换成了别的轴。

## 用户看到的「轴标题」从哪来？

- **`fvar.axisNameID`** 指向 **`name` 表**中某条字符串，这是规范里给轴起的可读名称（如「字重」「Weight」）。
- **`axisTag`（如 `wght`）** 只参与插值与语义，**不**直接作为 UI 标题显示。

因此：若全局常量名是「直线宽度」，且导出时未把轴展示名改成「字重」，则部分客户端会显示成常量名对应的 name 记录；要与 Photoshop 等里的叫法一致，请在导出对话框中填写期望的 **axis 展示名**（写入 `name` 表）。

## 实现说明

`addAxisNamesToTable`（`fontManager/tables/name.ts`）使用轴对象上的 **`name`**（兼容 **`axisName`**）写入 `name` 表；请保证导出管线传入的 `variants.axes` 上该字段正确。
